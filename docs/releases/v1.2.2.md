# ChatMem v1.2.2

v1.2.2 adds Favorites and several layout fixes on top of v1.2.1 summary-style migration. The release keeps full conversation migration, summary-style migration, trash, sync, ZCode, and MCP integration behavior intact.

## Highlights

- Added a conversation Favorites feature with star actions in the conversation list.
- Added a `收藏夹 / Favorites` entry in the bottom utility navigation.
- Favorites now open as a right-side workspace page, matching the Trash page behavior.
- Removed the visible `关于我们 / About us` utility entry from the bottom navigation.
- Restored the bottom utility navigation to the v1.2.1 one-line layout, aligned with the version label.
- Adjusted bottom utility button spacing so the Favorites count no longer truncates the `收藏夹 / Favorites` label.
- Moved the Settings page back button to a left-bottom floating control matching the home sidebar-collapse button.
- Bumped app/package versions to `1.2.2`.
- Updated the updater endpoint and README release links to `douxy1994/ChatMem`.

## User-Facing Changes

### Favorites

Users can now star important conversations from the left conversation list. A favorited conversation is stored as a lightweight snapshot:

- id
- source agent
- project path
- created time
- updated time
- summary/title

The snapshot does not copy full conversation content and does not change the original conversation storage location. Opening a favorite resolves back into the normal conversation-detail loading flow.

The Favorites page is shown in the right workspace, like Trash. This keeps the normal left navigation visible and avoids replacing the project/conversation tree with a special list.

### Bottom Navigation

The bottom utility navigation now contains:

- Favorites
- Trash
- Settings
- version label

`About us` is no longer visible in the utility navigation.

The buttons are laid out in one row and aligned with the version label, matching the v1.2.1 visual height.

Button widths now follow content instead of forcing a strict three-column split. Count badges stay visible without causing the Favorites label to collapse into an ellipsis.

### Settings Back Button

The Settings page no longer renders a text `Back / 返回` button in the page header. Instead, the close action is exposed as a left-bottom floating button with the same visual treatment as the home sidebar-collapse button.

The floating button keeps accessible labels and titles:

- English: `Back`
- Chinese: `返回`

Local Tauri note:

- The macOS `.app`, `.dmg`, and updater archive were generated.
- The local build ended at updater signing because `TAURI_PRIVATE_KEY` was not set.
- This does not block local `.app` installation, but GitHub release builds need `TAURI_PRIVATE_KEY` and `TAURI_KEY_PASSWORD` configured to publish updater assets.

Known broader test status:

- Focused v1.2.2 tests pass.
- Some older full-suite assertions may still expect previous UI labels or older settings label wiring. Those are separate from the Favorites and layout changes.

## Windows Parity

Windows developers should follow:

- `docs/windows-v1.2.2-favorites-implementation.md`

The Windows implementation is equivalent when:

- star/unstar writes `favoriteConversations` to settings
- favorites survive app restart
- favorites open in the right workspace
- normal left conversation navigation remains visible when Favorites is open
- the Favorites button toggles the Favorites workspace off when clicked again
- opening a normal conversation exits Favorites
- `About us` is not shown in the bottom utility nav
- bottom utility buttons align with the version label
- Settings uses the floating left-bottom return button
