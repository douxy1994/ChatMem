# ChatMem v1.2.1

v1.2.1 adds summary-style migration for cross-agent continuation while keeping the original full conversation migration and low-token toolbar prompt intact.

## Highlights

- Added a new `总结式迁移` option in the `Migrate` dialog.
- Kept `完整对话迁移` as the default migration path for copying or moving full conversations.
- Restored the conversation toolbar to its original low-token continuation prompt behavior.
- Added a source-backed continuation brief generator for compact handoff between agents.
- Added success feedback for summary-style migration: after `复制继续卡片`, ChatMem shows `继续卡片已复制` and closes the modal automatically.
- Bumped app/package versions to `1.2.1`.

## What Changed

### Summary-Style Migration

Open a conversation, click `Migrate`, then choose:

- `完整对话迁移`: writes a complete conversation copy or move to a target agent.
- `总结式迁移`: copies a compact continuation card to the clipboard without writing to a target platform and without deleting the source conversation.

The continuation card starts with `# Continuation Brief` and includes:

- repo root
- source conversation id
- active workline
- latest completed action
- canonical files
- obsolete context warnings
- focused continuation protocol
- evidence source pointers

### User Feedback

When `总结式迁移` succeeds:

1. The continuation card is copied to the clipboard.
2. A success toast appears.
3. The migration modal closes automatically.

If clipboard access fails, the modal stays open and ChatMem shows a failure notice.
