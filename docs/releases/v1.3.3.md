# ChatMem v1.3.3

v1.3.3 新增 Kimi Code 原生支持：Agent 集成（MCP + skill + 全局引导规则）和 Kimi Code 本地历史读取。macOS 版本先行发布，Windows 端按同步开发文档补齐。

## 背景

Kimi Code CLI 把全部运行时数据放在 `~/.kimi-code/`（可用 `KIMI_CODE_HOME` 迁移），用户级 MCP 配置是 `mcp.json`，用户级 skill 目录是 `skills/`，全局引导规则是 `AGENTS.md`，会话历史按 `sessions/<workDirKey>/<sessionId>/` 存放。ChatMem 按官方数据布局接入，不改动 Kimi Code 自身配置结构。

## 重点更新

### Kimi Code Agent 集成

- 设置 -> Agent 集成新增 `Kimi Code` 卡片，`一键安装到全部` 会同时安装 Kimi Code。
- 安装路径（默认数据根 `~/.kimi-code`，设置 `KIMI_CODE_HOME` 时跟随该变量）：
  - MCP 配置：`~/.kimi-code/mcp.json`，写入 `mcpServers.chatmem`
  - Skill：`~/.kimi-code/skills/chatmem/SKILL.md`
  - 全局规则：`~/.kimi-code/AGENTS.md`（托管块，幂等覆盖）
- MCP 配置形状遵循 Kimi Code 官方 `mcp.json` 结构：

```json
{
  "mcpServers": {
    "chatmem": {
      "command": "/Applications/ChatMem.app/Contents/MacOS/ChatMem",
      "args": ["--mcp"],
      "startupTimeoutMs": 30000
    }
  }
}
```

- 卸载只移除 ChatMem 写入的 `mcpServers.chatmem`、`skills/chatmem` 目录和 `AGENTS.md` 托管块，用户其他 MCP server、skill 和规则保持不变；历史和记忆数据不受影响。

### Kimi Code 本地历史

- 新增 `agentswap-kimi` adapter crate，读取：

```text
~/.kimi-code/sessions/<workDirKey>/<sessionId>/state.json
~/.kimi-code/sessions/<workDirKey>/<sessionId>/agents/main/wire.jsonl
~/.kimi-code/sessions/<workDirKey>/<sessionId>/agents/agent-N/wire.jsonl
```

- 从 `state.json` 读取标题、`workDir` 项目根目录和创建/更新时间。
- 从 `turn.prompt` 事件提取真实用户输入；从 `content.part` 事件提取 assistant 正文（`text`）和 thinking（`think`，存入 metadata）。
- 按 `(turnId, step)` 把同一推理步的 thinking、正文和 tool calls 聚合为一条 assistant 消息；`tool.result` 按 `toolCallId` 精确回填到对应调用。
- 主 agent 与子代理（agent-N）的 wire 都解析，按时间戳合并成同一时间线，子代理消息带 `kimi_agent` metadata 标记。
- 从 tool call 参数中的 `path` / `file_path` 等字段提取文件变化，从 `cwd` / `workdir` 等字段兜底项目根目录。
- adapter 为只读：不支持写入 Kimi Code 原生会话（与 Antigravity 一致）。
- 恢复命令：`kimi --session <sessionId>`。

### 来源检测与同步

- 首页来源下拉、WebDAV 上传、本地文件夹同步和 `import_all_local_history` 全部包含 `kimi`。
- 未安装 Kimi Code（无 `~/.kimi-code/sessions`）的机器不会在首页来源中显示 Kimi Code。

## 技术变更

- `crates/agentswap-core`
  - `AgentKind` 新增 `KimiCode`，`source_agent_name()` 返回 `Kimi Code`。
  - `tool_mapping` 把 Kimi Code 归入 Claude 工具族（`Bash` / `Read` / `Edit` / `Grep` / `Glob` 同名）。
- `crates/agentswap-kimi`（新 crate）
  - `KimiCodeAdapter` 读取 `state.json` + `wire.jsonl`，支持 `KIMI_CODE_HOME`。
  - 回归测试覆盖 wire 解析、子代理合并、标题回退和真实本机会话。
- `crates/agentswap-zcode/src/adapter.rs`
  - `write_conversation` 对 `KimiCode` 明确报只读错误。
- `src-tauri/src/agent_integration.rs`
  - `IntegrationAgent` 新增 `KimiCode`，`detect_agent_integrations` 返回 8 个 agent。
  - `install_agent_integration("kimi")` 写入 `mcp.json`、skill 和 `AGENTS.md`；`uninstall` 只移除 ChatMem 自己的配置项。
  - `KIMI_CODE_HOME` 仅在非测试构建中生效，单元测试始终使用临时 home。
  - 新增回归测试 `installs_kimi_code_with_mcp_skill_and_agents_md`。
- `src-tauri/src/main.rs`
  - `AGENT_KEYS`、`get_adapter`、`agent_key`、`agent_display_label` 注册 `kimi`。
- `src-tauri/src/chatmem_memory/sync.rs`
  - `LOCAL_HISTORY_AGENTS`、`get_adapter` 注册 `kimi`；新增 `resolve_kimi_storage_path`；`build_resume_command("kimi")` 返回 `kimi --session <id>`。
- `src-tauri/src/local_sync.rs`
  - `AGENT_LIST` 注册 `kimi`。
- `src/App.tsx` / `src/components/MigrateModal.tsx`
  - 来源下拉、agent 标签、中文标题、配置位置、handoff profile、关于页和迁移目标全部包含 Kimi Code。
- 版本号更新为 `1.3.3`：
  - `package.json`
  - `package-lock.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/Cargo.lock`
  - `src-tauri/tauri.conf.json`

## 验证

发布前执行：

```bash
cargo test --manifest-path crates/agentswap-kimi/Cargo.toml
cargo test --manifest-path src-tauri/Cargo.toml --bin chatmem installs_kimi_code_with_mcp_skill_and_agents_md
cargo test --manifest-path src-tauri/Cargo.toml --bin chatmem
cargo test --manifest-path src-tauri/Cargo.toml
npm run test:run
npm run build
git diff --check
npm run tauri -- build
```

已知环境差异：`chatmem_memory::repo_identity` 的 `canonical_repo_root_collapses_nested_paths_to_git_root` 在 macOS 上因 `/var` -> `/private/var` 符号链接失败，属 v1.3.3 之前就存在的本机环境问题，与本次改动无关。

macOS DMG 输出位置：

```text
src-tauri/target/release/bundle/dmg/ChatMem_1.3.3_aarch64.dmg
```

## Windows 同步开发

Windows 端同功能开发文档：

- `docs/windows-v1.3.3-kimi-implementation.md`

Windows 端达到同等功能时应满足：

- 设置 -> Agent 集成显示 `Kimi Code`，`Install all` 包含 Kimi Code。
- Kimi Code 路径使用 `%USERPROFILE%\.kimi-code\...`（或 `%KIMI_CODE_HOME%`）。
- 本地历史读取 `%USERPROFILE%\.kimi-code\sessions\<workDirKey>\<sessionId>\` 下的 `state.json` 和 `agents\**\wire.jsonl`。
- 首页来源下拉只在存在可读 Kimi Code 会话时显示 Kimi Code。
- 卸载只删除 ChatMem 写入的 `chatmem` MCP server、skill 目录和托管规则块。

## 已知限制

- macOS 包暂未做 Apple Developer ID 签名和 notarization，首次打开可能需要在系统设置中允许。
- 当前构建环境未配置 `TAURI_PRIVATE_KEY`，因此 updater 包签名会在 Tauri 打包末尾报错；`.app` 和 `.dmg` 产物不受影响。
- Kimi Code adapter 为只读，迁移到 Kimi Code 原生会话暂不支持（与 Antigravity 一致）。
