# ChatMem v1.3.1

v1.3.1 是 WebDAV 同步修复版本，继续沿用稳定的 Tauri + Rust + React 主线。

## 重点更新

### WebDAV 同步可靠性

- 修复 WebDAV 同步被本地历史中的陈旧 rollout 路径中断的问题。
- 当某条本地会话索引指向的源 transcript 已经不存在时，同步会跳过该陈旧条目并继续上传其他可用对话。
- 非文件缺失类错误仍会失败并提示，避免把解析错误或损坏数据静默吞掉。

### 工作台快捷同步

- 修复工作台“立即同步”提示“缺少 WebDAV 密码”的问题。
- 设置页验证 WebDAV 或执行 WebDAV 同步后，当前应用会话会缓存刚保存过的密码。
- 工作台快捷同步会先读取系统凭据；若系统凭据读取不到，但本次应用会话已保存过密码，则复用会话缓存继续同步。

### Windows 工作台体验

- 工作台右上角同步按钮改为云同步图标，同步中显示旋转与流动高光状态，避免用户误以为按钮无响应。
- Windows 顶部菜单栏移除 `View` 菜单，保留更精简的 ChatMem / File / Edit 菜单结构。

## 技术变更

- 版本号更新为 `1.3.1`：
  - `package.json`
  - `package-lock.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/Cargo.lock`
  - `src-tauri/tauri.conf.json`
- WebDAV 上传列表收集阶段新增陈旧本地会话识别：
  - `Rollout file not found`
  - `Conversation file not found`
  - `No such file or directory`
  - `os error 2`
- 前端新增 WebDAV 会话密码缓存，供设置页和工作台同步入口共享。
- 新增回归测试覆盖设置页保存密码后工作台快捷同步复用密码的场景。
- 新增工作台同步按钮 loading 状态测试，覆盖同步时的按钮状态、图标和结束后的复位。

## 验证

发布前执行：

```bash
npm run test:run -- src/__tests__/App.test.tsx src/__tests__/SyncSettings.test.tsx
cargo test --manifest-path src-tauri/Cargo.toml --bin chatmem
npm run build
git diff --check
npm run tauri -- build --bundles nsis,msi
```

macOS DMG 输出位置：

```text
src-tauri/target/release/bundle/dmg/ChatMem_1.3.1_aarch64.dmg
```

Windows 输出位置：

```text
src-tauri/target/release/bundle/nsis/ChatMem_1.3.1_x64-setup.exe
src-tauri/target/release/bundle/msi/ChatMem_1.3.1_x64_en-US.msi
```

## 已知限制

- macOS 包暂未做 Apple Developer ID 签名和 notarization，首次打开可能需要在系统设置中允许。
- 当前本地构建环境未配置 `TAURI_PRIVATE_KEY`，因此本地打包 Windows 产物时使用 `--bundles nsis,msi`，不会生成 updater 签名包。
