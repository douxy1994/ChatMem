# ChatMem v1.3.5

v1.3.5 正式打通 macOS / Windows 双端应用内自动更新，并修复 macOS 端安装更新的回退缺陷。

## 重点更新

### 双端应用内自动更新

- **更新密钥更换**：启用新的 minisign 密钥对，`src-tauri/tauri.conf.json` 的 `tauri.updater.pubkey` 已更新为新公钥。旧 Windows 构建机上的 `C:\Users\93219\.tauri\chatmem.key` 作废。
  - 注意：**v1.3.4 及更早版本内置的是旧公钥，无法验证新版本更新包签名**，请手动下载安装 v1.3.5 一次；此后版本即可应用内自动更新。
- **CI secrets 就位**：`douxy1994/ChatMem` 仓库已配置 `TAURI_PRIVATE_KEY` / `TAURI_KEY_PASSWORD`，CI 构建可产出签名更新包。
- **Release CI 触发修复**：`.github/workflows/release.yml` 新增 `release: published` 触发（保留 `v*` tag 触发），并统一解析 tag 名、按 tag 检出代码、加并发分组。此前 workflow 从未被触发过，发布 Release 后 CI 会自动追加全部平台资产：
  - Windows NSIS 安装包（`*_x64-setup.exe`）+ 签名更新包 + `.sig`
  - Windows 便携版 `ChatMem-v<version>-portable.zip`
  - macOS dmg（Apple Silicon / Intel）+ 签名更新包 `.app.tar.gz` + `.sig`
  - 双平台合并的 `latest.json`（应用内更新清单）

### macOS 安装更新回退修复

- `src-tauri/src/github_update.rs`：`install_github_release_update` 在非 Windows 平台直接返回明确错误（此前会继续下载 Windows `.exe` 再失败）。
- `src/updater/updater.ts`：识别该错误并回退到 Tauri 签名更新器（`installUpdate` + `relaunch`）。macOS 上「安装更新」从此走签名更新包，不再报 "does not include a Windows installer"。

### 文档

- `DEVELOPMENT.md` 重写：双端环境准备、新密钥位置与 GitHub Secrets、双端本地签名构建命令、标准发版步骤（Release 触发 CI）、双端更新机制说明、Windows 端开发与功能同步约定。

## 升级建议

- macOS：下载 `ChatMem-v1.3.5-macOS-Apple-Silicon.dmg`（或 Intel 版）手动覆盖安装一次；之后版本可应用内更新。
- Windows：本版本起 CI 首次产出 Windows 安装包，安装 `*_x64-setup.exe` 后，后续版本可应用内更新（Windows 走免密钥的 `.exe` 静默安装路径）。
