# ChatMem v1.3.2

v1.3.2 新增 Google Antigravity CLI 集成，继续保留 Gemini CLI 集成，不做破坏性重命名。

## 背景

Google 官方在 2026-05-19 宣布 Gemini CLI 向 Antigravity CLI 迁移。个人免费、Pro、Ultra 用户的 Gemini CLI 服务在 2026-06-18 停止，但企业授权和 Gemini Enterprise Agent Platform API Key 场景仍保留 Gemini CLI 访问。因此 ChatMem 采用新增目标而不是替换目标：

- `Gemini`：继续管理旧 Gemini CLI 的 MCP 和 `GEMINI.md` 引导。
- `Google Antigravity`：新增管理 Antigravity CLI 的 MCP、skill 和全局引导规则。

## 重点更新

### Agent 集成

- 设置 -> Agent 集成新增 `Google Antigravity` 卡片。
- `一键安装到全部` 会同时安装 Gemini CLI 和 Google Antigravity CLI，两者互不覆盖。
- `Google Antigravity` 的安装路径：
  - MCP 配置：`~/.gemini/antigravity-cli/mcp_config.json`
  - Skill：`~/.gemini/antigravity-cli/skills/chatmem/SKILL.md`
  - 全局规则：`~/.gemini/antigravity-cli/AGENTS.md`
- 卸载 `Google Antigravity` 只移除 ChatMem 写入的 MCP server、skill 和托管规则块，不删除用户其他 Antigravity 配置。

### 兼容性

- Gemini CLI 配置仍使用：
  - `~/.gemini/settings.json`
  - `~/.gemini/GEMINI.md`
- Antigravity CLI 配置独立使用：
  - `~/.gemini/antigravity-cli/mcp_config.json`
  - `~/.gemini/antigravity-cli/skills/`
- Gemini 和 Antigravity 可以并存。主界面来源下拉会运行时检测本机可读数据目录，只显示实际可用的来源；未安装 Gemini CLI 的机器不会显示 Gemini 来源。

### Antigravity 本地历史

- 新增 Antigravity 本地历史 adapter，读取：

```text
~/.gemini/antigravity/brain/<session>/.system_generated/logs/transcript.jsonl
```

- 解析 `USER_EXPLICIT` / `MODEL` / `SYSTEM` 事件为 ChatMem 对话消息。
- 从 `<USER_REQUEST>` 中提取干净的用户请求，不把 Antigravity metadata 混入标题。
- 从 tool call 参数中的 `Cwd` / `WorkingDirectory` / `ProjectPath` 等字段恢复真实项目根目录。
- 从 `AbsolutePath` / `FilePath` / `Path` 等字段提取文件变化。
- 保存 tool calls、thinking metadata、Antigravity event type/status，便于详情页和后续检索使用。

### 工作台来源与同步按钮

- 新增 `detect_conversation_sources` 命令，首页来源下拉只显示 `adapter.is_available()` 为真的本地来源。
- 工作台右上角“立即同步”按钮恢复循环箭头 icon 和禁用态旋转动画，不再使用云朵流光样式。

## 技术变更

- `src-tauri/src/agent_integration.rs`
  - `IntegrationAgent` 新增 `Antigravity`。
  - `detect_agent_integrations` 返回 7 个 agent。
  - `install_agent_integration("antigravity")` 写入独立 Antigravity CLI 配置和 skill。
  - `uninstall_agent_integration("antigravity")` 只移除 ChatMem 自己的配置项。
  - 新增回归测试 `installs_antigravity_cli_as_distinct_gemini_successor`。
- `crates/agentswap-gemini/src/adapter.rs`
  - `AntigravityAdapter` 读取 Antigravity brain transcript。
  - 新增 transcript 解析、项目根目录提取、文件变化提取和回归测试。
- `src-tauri/src/main.rs`
  - `AGENT_KEYS` 恢复 Gemini / Antigravity / OpenCode / ZCode / Hermes 的完整支持列表。
  - 新增 `detect_conversation_sources` 给首页来源下拉做可用性过滤。
- `src/App.tsx`
  - 主界面来源下拉改为运行时检测结果。
  - Gemini 保留为支持目标，但未安装时不在首页显示。
- 工作台发布助手更新到 `v1.3.2` release 文档和 Windows 同步开发文档。
- 版本号更新为 `1.3.2`：
  - `package.json`
  - `package-lock.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/Cargo.lock`

## Windows 同步开发

Windows 端同功能开发文档：

- `docs/windows-v1.3.2-antigravity-implementation.md`

Windows 端达到同等功能时应满足：

- 设置 -> Agent 集成显示 `Google Antigravity`，且不移除 `Gemini`。
- `Install all` 同时处理 Gemini CLI 和 Antigravity CLI。
- Antigravity CLI 的路径使用 `%USERPROFILE%\.gemini\antigravity-cli\...`。
- Antigravity 本地历史读取 `%USERPROFILE%\.gemini\antigravity\brain\<session>\.system_generated\logs\transcript.jsonl`。
- 首页来源下拉只显示当前机器实际可用的来源；未安装 Gemini CLI 时不显示 Gemini，已安装 Antigravity 时显示 Antigravity。
- 卸载只删除 ChatMem 写入的 `chatmem` MCP server 和托管规则块。

## 已知限制

- macOS 包暂未做 Apple Developer ID 签名和 notarization，首次打开可能需要在系统设置中允许。
- 当前构建环境未配置 `TAURI_PRIVATE_KEY`，因此 updater 包签名会在 Tauri 打包末尾报错；`.app` 和 `.dmg` 产物不受影响。
