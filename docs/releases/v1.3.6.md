# ChatMem v1.3.6

v1.3.6 修复 Windows 客户端可重复启动多个 GUI 进程的问题。

## Windows 单实例修复

- 桌面 GUI 现在使用 Tauri v1 官方 single-instance 插件，同一用户会话只保留一个 ChatMem GUI 进程。
- 再次双击桌面图标时，新进程立即退出，并显示、取消最小化和聚焦已经运行的主窗口。
- 关闭主窗口后仍按原有逻辑隐藏到系统托盘；再次启动 ChatMem 会重新显示该窗口。
- `ChatMem.exe --mcp` 在单实例插件初始化前运行，因此 Agent 集成使用的独立 MCP stdio 进程不受限制。

## 验收要求

- 正常启动后应只有一个 `ChatMem.exe` GUI 进程。
- 连续多次启动桌面快捷方式后，GUI 进程数量仍为一个。
- 主窗口隐藏到托盘后再次启动，原进程 PID 保持不变且窗口重新可见。
- `ChatMem.exe --mcp` 仍可完成 initialize 和 tools/list 握手。

## 发布与升级

- `v1.3.6` 使用仓库现有 Release CI 从同一个 tag 构建 Windows、macOS、Windows 便携版和签名更新包。
- Windows 推荐安装 `ChatMem_1.3.6_x64-setup.exe`；批量部署可使用 `ChatMem_1.3.6_x64_en-US.msi`。
- 已安装 v1.3.5 的用户可以通过应用内更新检查获取 v1.3.6。
