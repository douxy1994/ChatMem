# ChatMem v1.3.0

v1.3.0 是一次工作台与本地历史管理升级。这个版本继续使用稳定的 Tauri + Rust + React 主线，已移除未发布的原生重写实验代码。

## 重点更新

### 工作台首页

- 未选择对话时显示“工作台”，不再显示空白页。
- 工作台集中展示继续工作、收藏夹、项目时间线、推荐接续 agent、智能搜索、项目记忆、发布检查、对话质量、隐私清理和跨平台状态。
- 工作台标题、副标题、卡片布局和窄窗口自适应已重新整理，避免内容跑到其他卡片背后或在窄窗口错位。
- 在对话详情和本地历史详情中增加右下角浮层返回按钮，长对话中也可以直接回到工作台。
- 工作台右上角新增“立即同步”按钮，复用当前设置中的 WebDAV 或 OneDrive/本地同步配置，点击后直接同步并显示成功或失败提示。

### 收藏夹增强

- 收藏项支持备注、标签和置顶。
- 收藏夹入口和垃圾箱入口保持在右侧工作区打开，不替代左侧项目/对话树。
- 底部入口数量改为圆角数字徽标，避免“收藏夹”“垃圾箱”等文字被数字挤压省略。
- 收藏项可以复制“收藏继续卡片”，用于跨 agent 快速接续重要对话。

### 迁移与继续

- `Migrate` 弹窗保留“完整对话迁移”，并新增“总结式迁移”选项。
- 总结式迁移只复制 source-backed continuation brief，不向目标平台写入完整历史，也不删除原对话。
- 复制继续卡片后显示成功提示并自动关闭弹窗。

### 项目记忆与本地历史

- 项目记忆板块会根据当前项目加载启动规则、候选记忆、Wiki、checkpoint、handoff、run 和 artifact 状态。
- 点击对话里的“打开项目记忆视图”保持在当前工作台语境，不再误跳到本地历史空页。
- 本地历史扫描未匹配当前 repo root 时，提示信息改为诊断提示，方便检查项目路径或别名。

### 垃圾箱与同步删除

- 删除对话现在明确是“移入垃圾箱”。
- 对话在垃圾箱保留期内，OneDrive/本地同步副本不会立刻删除，确保仍可恢复。
- 只有清空垃圾箱或垃圾箱到期清理时，才会执行最终删除并清理已配置的同步副本。

### macOS 界面调整

- 设置入口移动到 macOS 菜单栏 `ChatMem -> 设置...`，左下角不再显示设置按钮。
- 菜单栏和底部栏图标重新整理，整理会话与管理分组使用不同图标。
- 对话中绿色背景消息里的链接改为白色下划线样式，避免蓝色链接在深色背景上看不清。
- 工具调用详情区域固定展开后的最大宽度，减少同类对话框忽大忽小。

## 技术变更

- 版本号更新为 `1.3.0`：
  - `package.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/tauri.conf.json`
- 设置数据模型扩展 `favoriteConversations`：
  - `note: string`
  - `tags: string[]`
  - `pinned: boolean`
- Rust 设置载荷同步扩展收藏夹字段，保证原生设置文件和前端 fallback 一致。
- Tauri 菜单新增 `open-settings` 事件，让 macOS 菜单栏设置入口进入应用设置页。
- 垃圾箱清理逻辑新增同步副本最终删除路径，移入垃圾箱阶段只保留恢复快照和警告信息。
- 工作台同步入口复用已有 Tauri command：
  - WebDAV：`sync_webdav_now`
  - OneDrive/本地同步目录：`sync_local_now`
  - 本地同步成功后重新加载会话列表，让下载到本机的新对话立即出现在工作台和侧边栏。

## Windows 同步开发

Windows 端同功能开发文档：

- `docs/windows-v1.3.0-workbench-implementation.md`

Windows 端达到同等功能时应满足：

- 工作台在未选择对话时默认显示。
- 工作台右上角提供即时同步按钮，并与设置页同步配置共享同一套命令和错误提示。
- 十个工作台板块均由本地数据派生，不依赖网络或占位假数据。
- 收藏夹备注、标签、置顶能持久化并在重启后恢复。
- 垃圾箱语义与 macOS 一致：移入垃圾箱不删除同步副本，清空或到期才最终删除。
- 设置入口遵守 Windows 平台习惯，可放在应用内设置页；macOS 使用菜单栏入口。
- Release Assistant 指向 `v1.3.0` 文档和 `douxy1994/ChatMem` 更新通道。
