# ChatMem v1.3.4

v1.3.4 修复 MCP 工具 schema 使用非标准 JSON Schema `uint` 格式的问题。

## 修复内容

- `src-tauri/src/chatmem_memory/models.rs`：所有暴露到 MCP schema 的 `usize` 计数字段添加 `#[schemars(with = "i64")]`，覆盖 `conversation_count`、`imported_conversation_count`、`skipped_conversation_count`、`scanned_conversation_count`、`linked_conversation_count`、`indexed_repo_count`、`total_message_count`、`returned_message_count`、`token_estimate`、`mention_count`、`dimensions`、`indexed_documents`、`fallback_indexed_documents`、`approved_memory_count`、`pending_candidate_count`、`search_document_count`、`indexed_chunk_count` 等。
- `src-tauri/src/chatmem_memory/runs.rs`：`RunRecord.artifact_count` 同样按 `i64` 输出 schema。
- `src-tauri/src/chatmem_memory/mcp.rs`：请求参数中的可选计数字段按 `Option<i64>` 输出 schema。

## 影响

- 序列化数据不变（JSON 数字原本就无符号区分），仅 schema 格式从非标准 `uint` 变为标准 `int64`，功能行为完全一致。
- 修复后 Kimi Code 等使用 ajv 校验的 MCP 客户端，启动加载 ChatMem 工具时不再打印 `unknown format "uint"` 警告。

## 构建说明

- 本版本由本地构建发布（macOS Apple Silicon），未配置 `TAURI_PRIVATE_KEY`，updater 签名包未生成。
